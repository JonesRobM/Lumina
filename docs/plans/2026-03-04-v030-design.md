# v0.3.0 Design: Periodic Structures, Multi-Shell Geometry, Anisotropic & Bianisotropic Materials, Substrate

**Date:** 2026-03-04
**Status:** Approved
**Scope:** `lumina-geometry`, `lumina-core`, `lumina-cli`, `lumina-gui`, `docs/`

---

## Problem

v0.2.x supports only finite, isolated assemblies of isotropic dipoles in a
uniform background medium. Three classes of physically important systems are
currently inexpressible:

1. **Periodic nanostructures** — nanoparticle arrays, metasurfaces, photonic
   crystal slabs — require a Bloch-periodic Green's function and k∥-dependent
   response.
2. **Core-shell and multi-layer particles** — Au@Ag, Au@SiO₂, and multi-shell
   MNPs — require per-dipole material assignment based on radial shell
   membership.
3. **Anisotropic and chiral materials** — nanorods, 2D materials, and chiral
   molecules — require a tensorial (3×3) polarisability α_EE and, for chiral
   systems, magneto-electric coupling tensors α_EM and α_MM.
4. **Substrate effects** — particles on glass, ITO, or silicon substrates —
   require the half-space Green's function correction.

Additionally, the documentation and tutorials have not been updated since
v0.1.3 and do not reflect the current SceneSpec TOML format or v0.2.x
capabilities.

---

## Goals

1. Ewald summation for 1D and 2D periodic arrays; k∥-dependent cross-sections
   and full dispersion maps ω(k∥).
2. `ShapeSpec::CoreShell` for concentric multi-shell MNPs on all applicable
   primitives.
3. Tensorial `Dipole` supporting isotropic, uniaxial, full-tensor, and
   bianisotropic (magneto-electric) polarisability.
4. Automatic solver reduction: 3N system when α_MM = 0 with corrected driving
   field; full 6N system otherwise.
5. Substrate image-dipole correction; substrate + periodic simultaneously
   supported.
6. Updated docs: theory chapters, SceneSpec TOML tutorial, worked examples.

---

## Non-Goals

- Sommerfeld integral substrate (rigorous half-space Green's function) —
  deferred to v0.4.
- DFT tensor import (VASP WAVEDER / Gaussian frequency-dependent α) — v0.4.
- 3D periodic structures (Ewald in all three directions) — v0.4.
- Nonlinear optics (SHG/THG) — v0.3.1 or v0.4.
- MPI / HPC distribution — v0.4.

#### 1.2 Ewald Decomposition (`lumina-core/src/solver/ewald.rs`)

For a 2D periodic array (lattice vectors **a**₁, **a**₂ in the xy plane) the
periodic dyadic Green's function at Bloch wavevector **k**∥ is:

```
G_per(r, k∥) = G_R(r, k∥) + G_G(r, k∥)
```

**Real-space sum** (converges as exp(−η²R²)):

```
G_R(r, k∥) = Σ_{R∈Λ} G_free(r − R) exp(ik∥·R) erfc(|r−R|η + k/2η)
```

**Reciprocal-space sum** (converges as exp(−G²/4η²)):

```
G_G(r, k∥) = (1/A_cell) Σ_{G} G̃(G+k∥, kz) exp(i(G+k∥)·ρ) exp(ikz|z|)
              × [1 + erf(kz/2η − η|z|)] / 2kz
```

where **G** runs over reciprocal lattice vectors, kz = √(k²−|G+k∥|²), ρ = (x,y),
and A_cell is the unit cell area.

For 1D chains, the reciprocal-space sum reduces to a 1D sum with modified
exponential-integral terms.

The optimal splitting parameter: η = √(π / A_cell) for 2D, giving equal
convergence rates for both sums.

```rust
pub struct EwaldGreens {
    pub lattice: LatticeSpec,
}

impl EwaldGreens {
    /// Full periodic dyadic Green's function at Bloch vector k_bloch (nm⁻¹).
    pub fn evaluate(
        &self,
        r: [f64; 3],       // r_i − r_j (displacement vector, nm)
        k_bloch: [f64; 3], // in-plane Bloch wavevector (nm⁻¹)
        k: f64,            // free-space wavenumber (nm⁻¹)
    ) -> [[Complex64; 3]; 3];

    /// Real-space contribution only (for testing convergence).
    pub fn real_space(&self, r: [f64; 3], k_bloch: [f64; 3], k: f64)
        -> [[Complex64; 3]; 3];

    /// Reciprocal-space contribution only.
    pub fn recip_space(&self, r: [f64; 3], k_bloch: [f64; 3], k: f64)
        -> [[Complex64; 3]; 3];
}
```

#### 2.2 Shell membership

Shell membership is determined at discretisation time by the **generalised
radial coordinate** of each dipole relative to the core surface:

| Base shape | Shell coordinate |
|---|---|
| Sphere | r = \|position\| |
| Cylinder | r = distance from axis |
| Ellipsoid | r = max(0, √((x/a)²+(y/b)²+(z/c)²) − 1) × min(a,b,c) |
| Cuboid | r = max(0, max(\|x\|/hx, \|y\|/hy, \|z\|/hz) − 1) × min(hx,hy,hz) |

Dipoles with r = 0 are in the core; 0 < r ≤ t₁ are in shell 1; etc.
Dipoles beyond the outermost shell fall back to the `ObjectSpec.material` value
(useful as a guard for boundary dipoles at coarse spacing).

`build_geometry()` sets `materials[i]` per-dipole from this assignment — the
wavelength-loop material resolver in CLI and GUI is unchanged.

### 3. Anisotropic and Bianisotropic Dipoles

#### 3.1 Extended `Dipole` type (`lumina-core/src/types.rs`)

```rust
pub struct Dipole {
    pub position: [f64; 3],
    /// Electric polarisability tensor α_EE (3×3 Complex64).
    /// For isotropic dipoles: α·I. For anisotropic: full tensor.
    pub alpha_ee: [[Complex64; 3]; 3],
    /// Magneto-electric coupling tensor α_EM.
    /// Zero for non-chiral materials.
    pub alpha_em: [[Complex64; 3]; 3],
    /// Magnetic polarisability tensor α_MM.
    /// Zero for non-magnetic materials (α_MM = 0 is the common case
    /// at optical frequencies).
    pub alpha_mm: [[Complex64; 3]; 3],
}
```

Constructors:

```rust
impl Dipole {
    /// Isotropic: α_EE = α·I, α_EM = α_MM = 0. Backwards compatible.
    pub fn isotropic(pos: [f64; 3], alpha: Complex64) -> Self;

    /// Uniaxial: α_EE = diag(α∥, α∥, α⊥) rotated to `axis`, others zero.
    pub fn uniaxial(
        pos: [f64; 3],
        alpha_par: Complex64,
        alpha_perp: Complex64,
        axis: [f64; 3],
    ) -> Self;

    /// Full electric tensor, no magnetic coupling.
    pub fn tensor(pos: [f64; 3], alpha_ee: [[Complex64; 3]; 3]) -> Self;

    /// Full bianisotropic: all three tensors specified.
    pub fn bianisotropic(
        pos: [f64; 3],
        alpha_ee: [[Complex64; 3]; 3],
        alpha_em: [[Complex64; 3]; 3],
        alpha_mm: [[Complex64; 3]; 3],
    ) -> Self;

    /// Returns true if any α_MM component is non-negligible.
    pub fn is_magnetic(&self) -> bool;

    /// Returns true if any α_EM component is non-negligible.
    pub fn is_chiral(&self) -> bool;
}
```

#### 3.2 Anisotropy TOML spec (`lumina-geometry/src/scene.rs`)

New optional field on `ObjectSpec`:

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type", rename_all = "lowercase")]
pub enum AnisotropySpec {
    /// Uniaxial: one optical axis, two independent polarisability components.
    Uniaxial {
        axis: [f64; 3],
        /// α⊥ = factor × α∥ (Clausius-Mossotti value).
        alpha_perp_factor: f64,
    },
    /// Full 3×3 tensor (real and imaginary parts separately).
    Tensor {
        re: [[f64; 3]; 3],
        im: [[f64; 3]; 3],
    },
    /// Bianisotropic / chiral Pasteur medium.
    Bianisotropic {
        axis: [f64; 3],
        alpha_perp_factor: f64,
        /// Chirality parameter κ (dimensionless).
        kappa_re: f64,
        kappa_im: f64,
        /// Magnetic ratio: α_MM / α_EE (0.0 for non-magnetic).
        magnetic_ratio: f64,
    },
}
```

`ObjectSpec` gains:

```rust
pub struct ObjectSpec {
    // ... existing fields ...
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub anisotropy: Option<AnisotropySpec>,
}
```

When `anisotropy` is `None`, the existing isotropic Clausius-Mossotti path is
used unchanged.

#### 3.3 Automatic solver reduction

The solver inspects the dipole set before choosing the system size:

```
any dipole has is_magnetic() → 6N×6N full bianisotropic system
any dipole has is_chiral()   → 3N system, corrected driving field
otherwise                    → 3N system, existing path (zero overhead)
```

**3N reduced bianisotropic** (α_MM = 0):
The corrected driving field at dipole i is:

```
E_eff,i = E_inc,i − ik·Σ_{j≠i} G_M(r_i, r_j) · α_EM_j · E_loc,j
```

where G_M = (1/k) ∇ × G_E is the magnetic Green's dyadic. This is iterated
within GMRES — the matvec gains one additional O(N²) term.

**6N full system**:
The state vector is (p₁,...,pN, m₁,...,mN). The 6N×6N matrix has 2×2 block
structure:

```
[ A_EE  A_EM ] [ p ]   [ E_inc ]
[ A_ME  A_MM ] [ m ] = [ H_inc ]
```

where A_EE is the existing 3N×3N matrix, and A_EM, A_ME, A_MM are built from
G_M coupling terms.

### 4. Substrate — Image Dipole Method

#### 4.1 `SubstrateSpec` (`lumina-core/src/solver/substrate.rs`)

```rust
    #[serde(default)]

    /// Substrate material identifier (must be a registered MaterialProvider).
    pub material: String,
}
```

TOML example:

```toml
[simulation.substrate]
z_interface = 0.0
material    = "SiO2_Palik"
```

#### 4.2 Image dipole Green's function

For a non-magnetic substrate with ε_sub(ω), the image contribution to the
system matrix is:

```
G_img(r_i, r_j) = diag(−Δε, −Δε, +Δε) · G_free(r_i, r_j')
```

where r_j' = (x_j, y_j, 2·z_interface − z_j) is the mirror position, and
the retarded reflection factor is:

```
Δε(ω) = (ε_sub − ε_env) / (ε_sub + ε_env)
```

(This is the quasistatic Fresnel factor. For the retarded correction, a phase
factor exp(2ik·z_j) modifies the z-component, included in the implementation.)

The substrate term is added to the off-diagonal blocks during assembly:

```
A_ij += −G_img(r_i, r_j)   (j ≠ i)
A_ii += −G_img(r_i, r_i')  (self-image term)
```

#### 4.3 Periodic substrate (forest on substrate)

When both `LatticeSpec` and `SubstrateSpec` are present, the image dipoles
also form a periodic array. The assembly runs two Ewald sums:

```
G_total(r_i, r_j, k∥) = G_Ewald(r_i − r_j, k∥)
                       + Δε_tensor · G_Ewald(r_i − r_j', k∥)
```

No new code paths required beyond calling `EwaldGreens::evaluate` twice with
the image positions.

## Testing Strategy

| Test | Validation target |
|---|---|
| Ewald vs brute-force sum (small truncation) | Convergence of real + reciprocal sums |
| Periodic chain → isolated dipole as a₁ → ∞ | Correct infinite-separation limit |
| 2D array extinction vs published Au array data | Physical accuracy |
| CoreShell sphere vs analytical Mie (coated sphere) | Shell geometry correctness |
| Uniaxial rod: C_ext(x-pol) ≠ C_ext(z-pol) | Anisotropy effect on spectra |
| Chiral dimer: non-zero CD | Bianisotropic coupling |
| Substrate: image dipole → enhanced near-field at z=0 | Correct reflection sign |
| Forest + substrate: spectra shift vs free-space | Combined path |
